S9 — App Shell & State · Interface Contract

Gantt.IO subsystem S9. Wires all subsystems together. Global application state, toolbar, view routing, file I/O orchestration, and event propagation. Dependencies: S1 (schema), S2 (storage adapter), S3 (worklog engine), S4 (task engine), S5 (canvas renderer), S6 (sidebar), S7 (edit drawer), S8 (work log UI). Used by: The application entry point (index.html + app.ts). No other subsystem uses S9 — S9 owns the composition root.

Files

  state.ts         ← Global state store (reactive, single source of truth)
  app.ts           ← AppShell class: orchestrator, toolbar, file menu, shortcuts
  view-router.ts   ← Switches between Gantt view (S5+S6) and List view (S6 alone)

  index.ts         ← Public API (createAppShell, AppShellAPI)

Core Responsibilities

  1. Global state management – holds current Project, work entries, UI settings (zoom, scroll, active view, filters).
  2. File orchestration – uses S2 to load/save project and work log files; triggers snapshot rebuild (S3 → S4) after load or mutation.
  3. View routing – renders either “Gantt” (S5 + S6 side‑by‑side) or “List” (only S6, full width).
  4. Toolbar & menus – provides UI controls for file operations, zoom, view switching, filter/sort/group, and search.
  5. Event propagation – when a task is edited via S7 or work log is added via S8, S9 triggers the required re‑renders (S5, S6, S7).
  6. Keyboard shortcuts – global shortcuts (Ctrl+S, Ctrl+O, Esc, etc.) and context‑aware handling.
  7. Scroll synchronisation – coordinates scroll positions between S5 and S6 so rows stay aligned.
  8. Persistent UI preferences – saves/restores collapsed groups, sort, filter, groupBy, zoom level, active view, etc. (e.g. via localStorage).

Global State Schema

Single reactive store. All mutations go through updateState(updater) to trigger subscriptions.

// src/s9-shell/state.ts

interface GlobalState {
  // Core data
  project: Project | null;           // null = no file loaded
  workEntries: WorkEntry[];          // always in sync with the work log file
  selectedTaskId: string | null;

  // UI view & layout
  activeView: "gantt" | "list";      // gantt = S5 + S6, list = only S6
  zoomLevel: number;                 // pixels per day (e.g. 20, 30, 50)
  scrollTop: number;                 // shared scroll position (used by both S5 & S6)

  // Sidebar UI preferences (persisted)
  sidebar: {
    collapsedGroupIds: Set<string>;
    sort: { field: SortField; direction: SortDirection };
    groupBy: GroupByOption;
    filterText: string;              // simple text search; more advanced filters can be derived
  };

  // Canvas additional state
  canvas: {
    hoveredTaskId: string | null;    // for potential tooltips
    dependencyVisibility: boolean;   // show/hide dependency arrows
  };

  // Work log UI
  workLogDialogOpen: boolean;
  workLogHistoryOpen: boolean;

  // Edit drawer
  editDrawerOpen: boolean;
  editDrawerTaskCopy: Task | null;   // mutable copy before save

  // File status
  currentProjectFile: File | null;   // reference to last opened file (for save)
  currentWorklogFile: File | null;
  isDirty: boolean;                  // true if unsaved changes exist
}

function defaultGlobalState(): GlobalState { ... }

AppShell API

The main entry point created by createAppShell(element: HTMLElement, options?: ShellOptions). It mounts the toolbar, container, and orchestrates all subsystems.

// src/s9-shell/app.ts

interface ShellOptions {
  defaultRowHeight?: number;          // pixels, passed to S5 & S6 (default 28)
  defaultZoom?: number;               // px/day (default 30)
  onProjectChanged?: (project: Project) => void;  // callback for telemetry/auto-save
}

interface AppShellAPI {
  // Lifecycle
  mount(): void;
  unmount(): void;

  // File operations (call S2, then rebuild state)
  newProject(): void;
  openProject(): Promise<void>;         // file picker → S2.loadProjectFile
  saveProject(): Promise<void>;
  saveProjectAs(): Promise<void>;
  openWorkLog(): Promise<void>;         // separate work log file (JSON/CSV)
  exportTasksToCSV(): Promise<void>;    // S2.exportTasksCSV

  // State access & mutation
  getState(): GlobalState;
  updateState(updater: (draft: GlobalState) => void): void;

  // Convenience actions (dispatch internally)
  setActiveView(view: "gantt" | "list"): void;
  setZoomLevel(pxPerDay: number): void;
  setSelectedTask(taskId: string | null, expandInSidebar?: boolean): void;
  openEditDrawer(taskId: string): void;
  closeEditDrawer(): void;
  openWorkLogDialog(): void;
  showWorkLogHistory(): void;

  // Subsystem access (for advanced use, usually not needed)
  getSidebarAPI(): SidebarAPI | null;
  getCanvasAPI(): CanvasAPI | null;

  // Keyboard shortcuts
  registerShortcut(key: string, handler: () => void, context?: string): void;
}

Orchestration Flows

1. Application startup

// Called by index.html after DOM ready
const shell = createAppShell(document.getElementById("app"), { defaultRowHeight: 28 });
shell.mount();

// Optionally load last project from localStorage or URL hash
if (storedProjectFileReference) shell.openProject();

2. Loading a project file (including work log)

async function loadProjectFile(file: File): Promise<void> {
  // 1. Load project JSON via S2
  const project = await FileLoader.loadProjectFile(file);
  // 2. If worklog_path is present or user selects companion file, load worklog
  let entries: WorkEntry[] = [];
  if (project.meta.worklog_path) {
    // S2.loadWorkLogFile based on extension
    entries = await FileLoader.loadWorkLogFile(worklogFileRef, 'json');
  }
  // 3. Deduplicate & rebuild snapshots (S3)
  entries = WorklogEngine.deduplicateEntries(entries);
  let updatedProject = WorklogEngine.rebuildSnapshots(project, entries);
  // 4. Rollup groups & compute conflicts (S4)
  updatedProject = Rollup.rollupGroups(updatedProject);
  updatedProject = Conflict.computeConflictStates(updatedProject, updatedProject.dependencies);
  // 5. Update global state
  shell.updateState(draft => {
    draft.project = updatedProject;
    draft.workEntries = entries;
    draft.currentProjectFile = file;
    draft.isDirty = false;
    draft.selectedTaskId = null;
  });
  // 6. Refresh S5 & S6 (they subscribe to state changes)
}

3. After task edit (S7 save)

// Called by S7 drawer when user clicks "Save"
async function handleTaskUpdate(editedTask: Task): Promise<void> {
  const oldProject = shell.getState().project!;
  // 1. Mutate via S4
  let newProject = updateTask(editedTask, oldProject);
  // 2. Rebuild snapshots (S3) — hours_spent unchanged, but completion_derived may change if estimated_hours changed
  const entries = shell.getState().workEntries;
  newProject = WorklogEngine.rebuildSnapshots(newProject, entries);
  // 3. Rollup groups & recompute conflicts (S4)
  newProject = Rollup.rollupGroups(newProject);
  newProject = Conflict.computeConflictStates(newProject, newProject.dependencies);
  // 4. Persist project file (S2)
  await FileSaver.saveProjectFile(newProject, shell.getState().currentProjectFile);
  // 5. Update state & notify subscribers
  shell.updateState(draft => {
    draft.project = newProject;
    draft.isDirty = false;
    draft.editDrawerOpen = false;
  });
}

4. Work log quick entry (S8)

async function addWorkEntry(entry: WorkEntry): Promise<void> {
  const state = shell.getState();
  const newEntries = WorklogEngine.addEntry(entry, state.workEntries);
  // Rebuild snapshots & rollup
  let updatedProject = WorklogEngine.rebuildSnapshots(state.project!, newEntries);
  updatedProject = Rollup.rollupGroups(updatedProject);
  updatedProject = Conflict.computeConflictStates(updatedProject, updatedProject.dependencies);
  // Persist both project and work log
  await FileSaver.saveProjectFile(updatedProject, state.currentProjectFile);
  await FileSaver.saveWorkLogFile(newEntries, state.currentWorklogFile, 'json');
  shell.updateState(draft => {
    draft.project = updatedProject;
    draft.workEntries = newEntries;
    draft.isDirty = false;
  });
}

5. Scroll synchronisation (S5 ↔ S6)

// When user scrolls the sidebar:
sidebar.on("scroll", (scrollTop) => {
  shell.updateState(draft => { draft.scrollTop = scrollTop; });
  // canvas listens to state.scrollTop and scrolls accordingly
});

// When user scrolls the canvas:
canvas.on("scroll", (scrollTop) => {
  shell.updateState(draft => { draft.scrollTop = scrollTop; });
  sidebar.setScrollTop(scrollTop);
});

Event Bus (Internal)

Used by S9 to decouple subsystems and avoid direct references between S5, S6, S7, S8. Subsystems publish events, S9 updates state and triggers re‑renders.

// src/s9-shell/event-bus.ts
type AppEvent =
  | { type: "TASK_SELECTED"; taskId: string | null }
  | { type: "TASK_UPDATED"; task: Task }
  | { type: "TASK_DELETED"; taskId: string }
  | { type: "WORKLOG_ADDED"; entry: WorkEntry }
  | { type: "WORKLOG_CSV_IMPORTED"; entries: WorkEntry[] }
  | { type: "VIEW_CHANGED"; view: "gantt" | "list" }
  | { type: "ZOOM_CHANGED"; zoom: number }
  | { type: "REQUEST_SAVE" };

const EventBus = {
  publish(event: AppEvent): void { ... },
  subscribe(handler: (event: AppEvent) => void): () => void { ... }
};

S9 subscribes to all events and executes the corresponding orchestration logic (e.g. TASK_UPDATED triggers the flow shown in section 3).

Toolbar & Menu UI

S9 is responsible for creating the toolbar DOM and binding actions. The toolbar includes:

  • File menu: New, Open, Save, Save As, Open work log, Export CSV.
  • View switcher: Gantt / List (toggle).
  • Zoom controls: slider or +/- buttons, range 10–100 px/day.
  • Filter/Sort/Group buttons: open dropdowns that modify state.sidebar (S6 reacts via subscription).
  • Search field: filters tasks by title/notes (sets state.sidebar.filterText).
  • Log work button: opens S8 quick‑entry dialog.

All interactions call AppShellAPI methods (e.g. setActiveView, setZoomLevel).

UI Persistence (localStorage)

On state changes that are user preferences (sidebar collapsed groups, sort, groupBy, zoom level, active view), S9 writes a serialised version to localStorage and restores it on next load.

function persistUIPreferences(state: GlobalState): void {
  const prefs = {
    collapsedGroupIds: Array.from(state.sidebar.collapsedGroupIds),
    sort: state.sidebar.sort,
    groupBy: state.sidebar.groupBy,
    zoomLevel: state.zoomLevel,
    activeView: state.activeView
  };
  localStorage.setItem("gantt-ui-prefs", JSON.stringify(prefs));
}

Error Handling & Recovery

SituationBehaviour
S2 load fails (invalid JSON, missing fields)Show error dialog with S2 error message; keep previous project if any, else empty state.
S4 mutation throws TypeError (invariant violation)Log error, show toast notification, revert to last known good state (no persistence).
Save fails (disk full, permission)Show error notification, set isDirty = true and retry option.
Cycle detected in dependenciesMark tasks in cycle with conflict_state = "error" (already handled by S4), show a warning banner in toolbar.
Work log CSV import has invalid rowsS2 throws SchemaValidationError; S9 displays row-level errors in a modal, allows user to correct or abort.

Integration of Subsystems (Dependency Map)

S9 imports and holds references to all subsystems after initialisation:

// Inside createAppShell:
const sidebar = createSidebar(config);   // S6
const canvas = createCanvas(config);     // S5
const drawer = createEditDrawer();       // S7
const worklogUI = createWorkLogUI();     // S8

// Subscribe to events from each subsystem
sidebar.on("select", (taskId) => shell.setSelectedTask(taskId));
canvas.on("barClick", (taskId) => shell.setSelectedTask(taskId));
drawer.on("save", (task) => handleTaskUpdate(task));
worklogUI.on("entryAdded", (entry) => addWorkEntry(entry));

// Pass shared state down on every change
shell.updateState.subscribe(() => {
  const state = shell.getState();
  if (state.activeView === "gantt") {
    canvas.update(state.project, state.zoomLevel, state.scrollTop);
  }
  sidebar.update(state.project, state.sidebar);
  if (state.editDrawerOpen) drawer.open(state.editDrawerTaskCopy);
});

Key Invariants

  • S9 is the only module that knows about all other subsystems (S2–S8). No circular dependencies.
  • All file I/O goes through S2; S9 never reads/writes localStorage for project data (only UI preferences).
  • After any mutation (task edit, work log addition), the following sequence runs: S3.rebuildSnapshots → S4.rollupGroups → S4.computeConflictStates before updating the global state.
  • Global state is the single source of truth – subsystems receive immutable data snapshots via their update() methods.
  • Scroll position is owned by global state; S5 and S6 synchronise via state.scrollTop and never directly call each other.
  • Keyboard shortcuts are unregistered when the shell is unmounted.
  • When a project is closed, S9 resets all subsystems and clears the state.

Caller Sequence – Full Example (User opens a project)

1. User clicks "File → Open" in toolbar.
2. S9 calls `shell.openProject()` → shows file picker.
3. S9 receives `File` object → calls `loadProjectFile(file)` internal.
4. loadProjectFile uses S2 to parse JSON → gets Project object.
5. S9 optionally asks for worklog file (if `worklog_path` exists, auto‑loads via S2).
6. S9 calls S3.deduplicateEntries, S3.rebuildSnapshots, S4.rollupGroups, S4.computeConflictStates in sequence.
7. S9 updates global state with the new project & entries.
8. Global state subscription fires → S5.update() with new project and zoom; S6.update() with new project & sidebar state.
9. Sidebar re‑renders the task tree; Canvas re‑draws the Gantt bars.
10. Toolbar displays project name, user can now edit.
