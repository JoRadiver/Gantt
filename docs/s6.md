# S6 — Sidebar Task List· Interface Contract
> Gantt.IO subsystem S6.
Renders the left-panel hierarchical task list with expand/collapse,
> selection, sorting, filtering, grouping, and scroll synchronization with the Gantt canvas.
> **Dependencies**: S1 (schema/types), S4 (task tree functions, color resolution, rollup data). > **Used by**: S9 (app shell — state management, scroll linking) and S5 (canvas — scroll synchronization). --- ## Files ``` src/s6-sidebar/ types.ts ← UI state types, sort/filter/group definitions tree-model.ts ← Converts flat tasks + collapsed state into renderable tree rows renderer.ts ← DOM/Virtual DOM rendering of the task list scroll-sync.ts ← Scroll position management and synchronization index.ts ← Public API (Sidebar class / factory) ``` --- ## Core Responsibilities ### 1. **Tree Rendering** - Takes a flat `Task[]` from the project and a set of collapsed group IDs. - Builds a depth‑first ordered list of *visible* rows, applying: - **Filtering**: Hide tasks that do not match the current filter predicate. - **Sorting**: Reorder siblings according to the current sort field/direction. - **Grouping** (optional): Insert group header rows when a grouping field is active. - Renders each row with: - Indentation proportional to tree depth. - Type badge (`task`, `group`, `milestone`). - Task title (monospace, same as canvas label). - Mini progress bar (width = `completion_active * 100%`, using resolved color pastel/mid). - Expected hours (formatted with one decimal). - People pips: up to two initials circles; overflow indicator `+N`. - Row height must be identical to canvas bar row height (set by S9 configuration). - Supports keyboard navigation (arrow keys, enter to expand/collapse, space to select). ### 2. **Expand / Collapse** - Groups can be expanded or collapsed. - State is stored as a `Set<string>` of collapsed group IDs. - Collapsing a group hides all its descendants. - Events `onRowCollapse(task_id)` and `onRowExpand(task_id)` notify S9. ### 3. **Selection** - Single task selection (multi‑select is planned but not required for first implementation). - Selected row is highlighted with a distinct background. - Event `onRowSelect(task_id, event)` is emitted when a row is clicked or keyboard‑selected. ### 4. **Sort, Filter, Group** - **Sort**: - By `title`, `start_date`, `end_date`, `estimated_hours`, `completion_active`, `prio`. - Direction `asc` or `desc`. - **Filter**: - User‑defined predicate (e.g. “only tasks assigned to me”, “priority is critical”, “title contains X”). - Provided as a function `(task: Task, ancestors?: Task[]) => boolean`. - Hidden tasks are completely omitted from the tree; their parent groups are shown only if they have any visible descendants. - **Group** (optional): - Group by `assignee` (first person) or `priority` or `type`. - Inserts non‑selectable group header rows that can be collapsed/expanded independently. - Group headers are not tasks – they have no dependencies nor edit drawer. - S6 does **not** mutate the project. All sorting/filtering/grouping are ephemeral UI transformations. ### 5. **Scroll Synchronization** - Maintains `scrollTop` of the sidebar container. - Provides `getScrollTop()` and `setScrollTop(value)` methods for S5 and S9 to sync with the canvas. - When the user scrolls the sidebar, emits `onScroll(scrollTop)` so S5 can move the canvas accordingly. - Row height is fixed and known to both S5 and S6; scroll position can be accurately converted to the first visible row index. ### 6. **Events & External Control** - Emits: - `onRowSelect(task_id)` - `onRowCollapse(task_id)` - `onRowExpand(task_id)` - `onScroll(scrollTop)` - `onStateChange(newState)` (for S9 to persist UI preferences) - Accepts external commands: - `setSelection(task_id)` - `expandAll()`, `collapseAll()` - `expandTo(task_id)` – recursively expand all ancestors so the task becomes visible. - `setFilter(predicate)`, `setSort(field, direction)`, `setGroupBy(groupByOption)` --- ## Types ### UI State (maintained by S6, persisted by S9 if desired) ```ts type SortField = "title" | "start_date" | "end_date" | "estimated_hours" | "completion_active" | "prio" type SortDirection = "asc" | "desc" type GroupByOption = "none" | "assignee" | "priority" | "type" interface SidebarState { collapsedGroupIds: Set<string> // IDs of groups that are collapsed selectedTaskId: string | null scrollTop: number sort: { field: SortField; direction: SortDirection } filterPredicate: ((task: Task, ancestors: Task[]) => boolean) | null groupBy: GroupByOption } ``` ### Row Types (for rendering) ```ts type RowType = "task" | "group" | "milestone" | "group-header" interface RenderRow { id: string // task_id for task/group/milestone; synthetic id for group-header type: RowType depth: number // 0 for root-level items task?: Task // present for task/group/milestone rows groupHeaderValue?: string // e.g. "Critical", "John Doe" for group-header rows isCollapsible?: boolean // true for group rows and group-header rows isExpanded?: boolean // relevant only if isCollapsible isSelected?: boolean indentPx: number // precomputed depth * INDENT_WIDTH } ``` ### Render Configuration ```ts interface SidebarConfig { rowHeight: number // pixels, must equal canvas row height indentWidth: number // pixels per depth level (e.g., 20) containerElement: HTMLElement // where the sidebar will be mounted } ``` ### Public API (returned by `createSidebar`) ```ts interface SidebarAPI { // Lifecycle mount(): void destroy(): void // Rendering (called when project or UI state changes) update(project: Project, state: SidebarState): void // State queries & mutations getState(): SidebarState setState(partial: Partial<SidebarState>): void setSelection(task_id: string | null): void expandAll(): void collapseAll(): void expandTo(task_id: string): void // expands all ancestors of the given task // Scroll control (for canvas sync) getScrollTop(): number setScrollTop(scrollTop: number): void // Convert a point (e.g., click) to a task ID (for hit testing from canvas) hitTest(y: number): string | null // Events (register callbacks) on(event: "select", callback: (taskId: string, event: MouseEvent) => void): void on(event: "collapse", callback: (taskId: string) => void): void on(event: "expand", callback: (taskId: string) => void): void on(event: "scroll", callback: (scrollTop: number) => void): void on(event: "stateChange", callback: (newState: SidebarState) => void): void } ``` --- ## Functions (`tree-model.ts`) These pure functions convert the raw tasks + UI state into renderable rows. ```ts // Builds the list of RenderRow objects respecting collapse, sort, filter, group. // Uses S4.tree.getChildren, getDescendants, getAncestors, getRoots. function buildTreeRows( tasks: Task[], state: SidebarState ): RenderRow[] // Sorts an array of sibling tasks according to the current sort settings. function sortSiblings( siblings: Task[], field: SortField, dir: SortDirection ): Task[] // Filters tasks recursively: a task is shown if it passes the predicate OR // any of its descendants passes (to keep the parent visible). function filterTree( tasks: Task[], predicate: (task: Task, ancestors: Task[]) => boolean, ancestors: Task[] = [] ): Task[] // For grouping: inserts synthetic group-header rows before each group of tasks. // Returns a flat array that interleaves group headers and tasks. function applyGrouping( rows: RenderRow[], groupBy: GroupByOption, tasks: Task[] ): RenderRow[] ``` --- ## Scroll Synchronization (`scroll-sync.ts`) Exported utilities to help S9 and S5 keep the two scroll positions aligned. ```ts // Converts a scrollTop (pixels) to the index of the first visible row. function scrollTopToRowIndex(scrollTop: number, rowHeight: number): number // Converts a row index to the scrollTop needed to bring that row to the top. function rowIndexToScrollTop(rowIndex: number, rowHeight: number): number ``` S6 uses the above internally, but they are exported so that S5 can perform the same calculation for its own row indexing (if needed). S6 and S5 share the same `rowHeight` value. --- ## Error Handling | Condition | Behaviour | |---|---| | `containerElement` not found when mounting | Throws `DOMError` with message "Sidebar container missing" | | `project.tasks` is empty | Renders a single row: "No tasks" (non‑selectable) | | Filter predicate throws an exception | Logs error to console, treats the task as **not passing** the filter | | `expandTo` called with a task that does not exist or is not a descendant of any collapsed group | No effect (silent) | | Sorting by `completion_active` when both `completion_manual` and `completion_derived` are missing | Falls back to `0.0` | S6 never throws unrecoverable errors for invalid input; it degrades gracefully by showing an empty or partially filtered tree. --- ## Dependencies (explicit imports) - `S1`: `Task`, `Project`, `PaletteKey`, `getColorStop` (for mini progress bar). - `S4`: `getChildren`, `getRoots`, `getAncestors`, `resolveColor` (for task color determination, used by mini‑bar and type badge? Only mini‑bar uses resolved color; badge uses `type`). - No other S‑subsystems. S6 **does not** import S2, S3, S5, S7, S8, S9. --- ## Key Invariants | Rule | Where enforced | |---|---| | Row height is identical to canvas row height (set by S9 config) | `SidebarConfig.rowHeight` | | Collapsed groups hide all descendants, but their own row remains visible | `buildTreeRows` | | Filtering applies to a task if it or any descendant passes | `filterTree` | | Group‑header rows are never selectable, not part of the real task list | `applyGrouping` sets `type: "group-header"` | | S6 does not mutate the project or worklog | throughout – read‑only UI | | S6 does not perform any date arithmetic or dependency resolution | uses S4 for tree queries only | | Selection is stored as a single `string | null`; multi‑select is not in the contract | `SidebarState.selectedTaskId` | | `expandTo` expands ancestors but does not change selection | `SidebarAPI.expandTo` | --- ## Caller Sequences (for reference – not part of S6) ### Initialisation (S9) ``` const config = { rowHeight: 28, indentWidth: 20, containerElement: document.getElementById("sidebar") } const sidebar = createSidebar(config) sidebar.on("select", (taskId) => { shell.setSelectedTask(taskId) }) sidebar.on("collapse", (groupId) => { shell.saveCollapsedState(groupId, true) }) sidebar.on("scroll", (scrollTop) => { canvas.setScrollTop(scrollTop) }) sidebar.mount() sidebar.update(project, initialSidebarState) ``` ### After project change (S9) ``` const newState = sidebar.getState() newState.scrollTop = canvas.getScrollTop() // keep scroll synced if canvas moved sidebar.update(updatedProject, newState) ``` ### When user clicks a canvas bar (S5 → S9 → S6) ``` // S9 receives taskId from canvas hit test sidebar.setSelection(taskId) sidebar.expandTo(taskId) // ensure the task is visible // sidebar automatically scrolls to the selected row? Not required; selection highlight alone is enough. ``` ### Persisting UI state (S9 on beforeunload) ``` const uiPrefs = { collapsed: Array.from(sidebar.getState().collapsedGroupIds), sort: sidebar.getState().sort, groupBy: sidebar.getState().groupBy, // filterPredicate is not persisted (user-defined function) } localStorage.setItem("gantt-ui", JSON.stringify(uiPrefs)) ``` --- ## Notes - **Rendering approach**: The first implementation can use plain DOM manipulation (create/update/remove rows). Later a virtual DOM (e.g., lit‑html or React) may be introduced, but the public API remains the same. - **Performance**: For projects with hundreds of tasks, `buildTreeRows` should run in O(n) using memoised `collapsedGroupIds` and a single pass over the flattened task list (converted to a map from id to children). Sorting is applied only at sibling level – the global order is depth‑first with sorted siblings. - **Accessibility**: Render rows with `role="treeitem"` and proper ARIA attributes for expand/collapse. Keyboard navigation is part of the contract. - **Future extensions**: Multi‑select (Shift+Click, Ctrl+Click) can be added without changing the public API – only the `selectedTaskId` would become `Set<string>` and the `onSelect` event would carry multiple IDs. This document describes the single‑select baseline.
